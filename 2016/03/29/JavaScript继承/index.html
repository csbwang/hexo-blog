JavaScript 继承

JavaScript是通过函数实现的伪类，函数没有函数签名，在JavaScript中，继承主要通过原型链实现的，其主要思想是利用原型让一个引用类型继承另一个引用类型的属性和方法。

构造函数、原型、实例的关系

每个构造函数都有一个原型对象，原型对象都包含一个指向构造函数的指针，而每个实例都包含一个指向原型对象的内部指针，如下图所示：

原型链的产生

假如我们改变构造函数的原型对象，让原型对象等于另一个类型的实例（也就是改变当前实例中的指向原型对象的指针，让其指向另外一个类型的实例），此时的原型对象作为另一个类型的实例将包含一个指向另一个原型对象的指针，而另一个原型对象中也包含着一个指向构造函数的指针，假如另一个原型又是另一个类型的实例，这样层层递进就形成原型链了，所有的引用类型都默认继承了Object的实例，如下图所示：

如果要在子类中重写父类方法，一定要在继承之后进行重写。原型继承还存在以下两个问题：

  1. 由于包含引用类型的原型属性会被所有实例共享，所以一般我们将包含引用类型的属性定义在构造函数中（非引用类型属性不受影响）。但是在原型继承时，原型对象实际上会变成另外一个类型的实例，这样实例属性也就升级为原型属性了，包含引用的属性就会被所有实例所共享。
  2. 在创建子类的实例时，如果超类型构造函数包含参数时，不能像超类型的构造函数中传递参数。

组合继承

将原型链和借用构造函数的技术结合起来，使用原型链实现对原型属性和方法的继承，而通过借用构造函数来实现对实例属性的继承。

组合继承避免了原型链和借用构造函数的缺陷，融合他们的优点，是JavaScript中比较常用的继承实现方式。但是这种方式也有自己的不足，它最大的问题是需要调用两次超类构造函数，第一次在子类创建原型时，第二次是在子类构造函数内部，并且两次创建出超类对象的所有实例属性，第一次创建的全部实例属性在子类创建的原型中，第二次创建的全部实例属性称为子类实例的全部属性，如下图所示：
