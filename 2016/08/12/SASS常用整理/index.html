SASS

SASS是一种CSS的预编译语言，可以方便快速的编写CSS，本文总结SASS的常见用法，这些常见用法基本可以满足日常大部分的使用场景，其他用法可以参照官方文档。

SASS安装

  • SASS的安装需要Ruby环境，先安装Ruby
  • 如果安装好了Ruby，执行下面命令：gem install sass即可安装

SASS提供四个编译风格的选项：

  • nested：嵌套缩进的css代码，它是默认值。
  • expanded：没有缩进的、扩展的css代码。
  • compact：简洁格式的css代码。
  • compressed：压缩后的css代码。

最后一种方式产生的css代码可读性很差，一般在产品上线时采取这种方式

你也可以让SASS监听某个文件或目录，一旦源文件有变动，保存一下，就自动生成编译后的版本。

注释

SASS共有两种注释风格。
标准的CSS注释 / comment / ，会保留到编译后的文件。
单行注释 // comment，只保留在SASS源文件中，编译后被省略。
在/*后面加一个感叹号，表示这是”重要注释”。即使是压缩模式编译，也会保留这行注释，通常可以用于声明版权信息。

代码的重用

混入(Mixin)允许你定义可以在整个样式表中重复使用的样式，而避免了使用无语意的类（class)，mixin有点像C语言的宏（macro），是可以重用的代码块。

混入有一下四个特点：

  1. 可以定义参数，并且参数可以设置默认值
  2. 混入也可以包含选择器和属性的混合体，甚至可以包含&引用
  3. 混入（mixin）定义也可以包含其他的混入
  4. 混入的使用必须放在定义的后面

继承@extend

设计一个页面时常常遇到这种情况：当一个样式类（class）含有另一个类的所有样式，并且它自己的特定样式。处理这种最常见的方法是在HTML同时使用一个通用样式类和特殊样式类，这样不利于类的维护，@extend指令避免这些问题，告诉 Sass 一个选择器的样式应该继承另一选择器，它可以扩展id选择器和类选择器。

导入文件

@import 需要一个文件名来导入。默认情况下，它会寻找一个 Sass 文件直接导入，但在以下情况下， 仅作为普通的 CSS @import 规则语句，不会导入任何 Sass 文件，导入时可以同时导入多个文件，文件之间采用逗号分隔。

  • 如果文件的扩展名是 .css。
  • 如果文件名以 http:// 开始。
  • 如果文件名是 url()。
  • 如果@import 中包含任何的媒体查询（media queries）。

如果你有一个 SCSS 或 Sass 文件要导入，但不希望将其编译到一个CSS文件，你可以在文件名的开头添加一个下划线。这将告诉Sass不要将其编译到一个正常的CSS文件。然后，在导入语句中却不需要添加下划线。例如，你可能有一个命名为 _colors.scss 的文件，但是不会编译成 _colors.css 文件。你可以这么做
